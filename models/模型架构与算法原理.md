# 草莓三维姿态估计模型架构与算法原理

## 1. 算法前向传播过程 (学术伪代码)

**Algorithm 1: Forward Pass of Strawberry Pose Estimation Network**

**Input:** Monocular Image $I \in \mathbb{R}^{B \times 3 \times H_{in} \times W_{in}}$  
**Output:** Global Parameter Vector $Y \in \mathbb{R}^{B \times 22}$

1.  **Feature Extraction (Backbone):**  
    $F_{base} = \text{Darknet19}(I)$  // $F_{base} \in \mathbb{R}^{B \times 1024 \times 18 \times 25}$
2.  **Coordinate Augmentation (CoordConv):**  
    $C_x, C_y = \text{GenerateCoordinates}(H, W)$  
    $F_{coord} = \text{Concat}([F_{base}, C_x, C_y])$  
    $F_{reduced} = \text{LeakyReLU}(\text{BN}(\text{Conv}_{1\times1}(F_{coord})))$
3.  **Global Context Encoding (Transformer Neck):**  
    $X_{seq} = \text{Flatten}(F_{reduced}) + PE_{2D}$ // Add Sine-Cosine Position Embedding  
    $X_{attn} = \text{TransformerEncoder}(X_{seq})$  
    $F_{neck} = \text{Reshape}(X_{attn})$ // $F_{neck} \in \mathbb{R}^{B \times 256 \times 18 \times 25}$
4.  **Information Aggregation:**  
    $V_{global} = \text{GlobalAveragePool}(F_{neck})$ // $V_{global} \in \mathbb{R}^{B \times 256}$
5.  **Decoupled Task Prediction (Head):**  
    $Y_{reg} = \text{MLP}_{reg}(V_{global})$ // Output 21-dim (2D center, 8 vertices, 3D dims)  
    $Y_{conf} = \text{MLP}_{conf}(V_{global})$ // Output 1-dim (Confidence score)
6.  **Final Concatenation:**  
    $Y = [Y_{reg}, Y_{conf}]$
7.  **Return** $Y$

## 2. 核心计算逻辑数学抽象

### (1) 归一化坐标通道 (CoordConv)
为了打破卷积神经网络的平移不变性，增强对绝对位置的感知，引入标准化坐标通道：
$$C_{x,i,j} = \frac{2j}{W-1} - 1, \quad C_{y,i,j} = \frac{2i}{H-1} - 1$$
其中 $i, j$ 分别为特征图的行与列索引。

### (2) 2D 正余弦位置编码 (Positional Encoding)
Transformer 模块通过 2D Sine-Cosine 编码注入空间先验：
$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right), \quad PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$
其中 $d$ 为嵌入维度，$pos$ 分别取横向与纵向索引，最后拼接形成 $PE_{2D}$。

### (3) 多头自注意力机制 (Self-Attention)
Transformer Neck 核心计算逻辑，通过全局关联建模草莓特征：
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

### (4) 目标回归参数向量
网络输出层 $Y$ 的物理含义定义为：
$$Y = [u_c, v_c, \underbrace{\Delta u_1, \Delta v_1, \dots, \Delta u_8, \Delta v_8}_{\text{8 Vertices Offsets}}, \underbrace{l, w, h}_{\text{3D Size}}, c]$$

## 3. 架构设计优势分析

本系统采用 **Darknet-19 + Transformer Neck + Decoupled Head** 的耦合架构，旨在解决单目图像中草莓目标尺度变化大、空间位置敏感及姿态多变的问题。

首先，**Darknet-19** 作为主干网络，以较低的计算成本提供了高效的局部特征提取能力，确保了系统的实时性。然而，传统 CNN 受限于感受野大小，难以建模草莓表面纹理与全局几何轮廓之间的长程依赖。为此，本设计引入了 **Enhanced Transformer Neck**，利用全局自注意力机制捕捉图像各区域间的关联，显著增强了网络对复杂遮挡环境下草莓整体姿态的推断能力。

此外，**CoordConv** 与 **2D 位置编码** 的引入，弥补了 CNN 在处理回归任务（如中心点像素坐标）时对绝对空间位置感知不足的缺陷。最后，**解耦头（Decoupled Head）** 结构将几何参数回归与目标置信度预测任务分离，避免了多任务训练过程中的梯度干扰。这种“局部提取-全局增强-任务解耦”的设计方案，使模型不仅能精确提取草莓的细微形态特征，更能稳定地回归出高精度的三维姿态与尺寸参数，满足农业机器人精准作业的需求。
