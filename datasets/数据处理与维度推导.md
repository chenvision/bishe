# 数据处理流程与维度推导

## 1. 模型数据张量维度 (Tensor Shape) 推导过程

假设输入 Batch Size 为 $B$，输入图像分辨率为 $800 \times 600$。

| 阶段 | 模块/算子 | 输出维度 (Shape) | 关键变换说明 |
| :--- | :--- | :--- | :--- |
| **输入** | Image Tensor | $(B, 3, 600, 800)$ | 图像经过 `Resize` 与 `ToTensor` 转换 |
| **Backbone** | Block 1 (Conv + Pool) | $(B, 32, 300, 400)$ | $3 \times 3$ 卷积降维，最大池化下采样 |
| | Block 2 (Conv + Pool) | $(B, 64, 150, 200)$ | 下采样，特征通道增至 64 |
| | Block 3 (Conv $\times 3$ + Pool) | $(B, 128, 75, 100)$ | $1 \times 1$ 瓶颈层 + $3 \times 3$ 特征融合 |
| | Block 4 (Conv $\times 3$ + Pool) | $(B, 256, 37, 50)$ | 空间分辨率进一步压缩 |
| | Block 5 (Conv $\times 5$ + Pool) | $(B, 512, 18, 25)$ | 深度特征提取，进入高层语义空间 |
| | Block 6 (Conv $\times 5$) | $(B, 1024, 18, 25)$ | 获取全局语义特征图 |
| **Neck** | CoordConv (AddCoords) | $(B, 1026, 18, 25)$ | 拼接标准化 $(x, y)$ 坐标通道 |
| | Reduce Conv | $(B, 256, 18, 25)$ | 特征降维，适配 Transformer 嵌入维度 |
| | Flatten & PE | $(B, 450, 256)$ | 序列化 ($18 \times 25=450$)，注入 2D 位置编码 |
| | Transformer Encoder | $(B, 450, 256)$ | 多头自注意力机制进行全局建模 |
| | Reshape | $(B, 256, 18, 25)$ | 还原为空间特征图格式 |
| **Head** | Global Average Pool | $(B, 256, 1, 1)$ | 空间维度压缩，聚合全局上下文 |
| | Flatten | $(B, 256)$ | 展平为一维特征向量 |
| **输出** | Decoupled Branches | $(B, 22)$ | 回归 21 维几何参数 + 1 维置信度 |

## 2. 实验设置与数据预处理描述

### 2.1 数据预处理 (Data Preprocessing)
本实验中，针对单目图像的 6D 姿态估计任务，设计了严谨的数据预处理流水线：
*   **图像归一化**：将原始 RGB 图像各通道像素值除以 255.0，归一化至 $[0, 1]$ 范围，以加速模型收敛并稳定训练梯度。
*   **几何投影变换**：利用相机内参矩阵（$f_x=400.32, f_y=400.32, c_x=400.0, c_y=300.0$）及欧拉角转换得到的旋转矩阵，将草莓 3D 物体坐标系下的 8 个立方体顶点及 1 个几何中心点投影至 2D 像素平面。
*   **标签编码 (Label Encoding)**：采用相对坐标回归策略。目标向量（Target Vector）被编码为 22 维，包含：
    *   **中心点偏移量**：草莓 2D 中心相对于网格左上角的归一化偏移 $(dx, dy)$。
    *   **顶点残差**：8 个 2D 投影顶点相对于中心点的像素偏移量 $(\Delta u_i, \Delta v_i)$。
    *   **3D 物理参数**：草莓的长、宽、高三维物理尺寸 $(l, w, h)$。
    *   **置信度**：目标存在性的二值置信度评分。

### 2.2 数据增强与实验配置 (Augmentation & Setup)
为了增强模型在复杂农业环境下的鲁棒性，系统采用了以下策略：
*   **在线增强 (Online Augmentation)**：引入 `Albumentations` 增强库。实验中对图像执行随机亮度/对比度调整、颜色抖动以及几何缩放等操作。由于任务涉及 2D 坐标回归，所有空间变换均与投影关键点（Keypoints）保持同步更新，确保图像与标签的几何一致性。
*   **数据划分**：将 `Straw6D` 原始数据集按照预设比例划分为训练集与验证集，输入图像统一重采样至 $800 \times 600$ 像素，以平衡检测精度与端到端推理速度。
